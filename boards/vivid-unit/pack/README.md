# Board pack assets for raw-image release flow

This directory is now only used by the **raw image** pack stage.

## Release target

VUOS now treats the GPT raw disk image as the only release artifact.
The pack stage no longer builds:

- `embedded-update.img`
- `update.img`
- RKAF / RKFW containers
- `package-file`
- `parameter.txt`
- `partition-metadata.txt`

### Notes on partition layout

The raw GPT image now contains these partitions only:

1. `uboot-env`
2. `boot`
3. `rootfs`

So the Linux root filesystem is now `/dev/mmcblk0p3`.

## Bootloader inputs

The raw image uses the bootloader generated by the U-Boot stage:

- `out/uboot/<target>/u-boot-rockchip.bin`

That image is written into the raw disk image at the fixed Rockchip boot offset.

The U-Boot stage now prefers the vendor BL31 kept in:

- `boards/<target>/rkbin/rk3399_bl31_v1.36.elf`

If that file is absent, you may still override it manually with:

- `BL31=/path/to/bl31.elf`

TF-A remains available as a manual test stage, but it is no longer started automatically by the normal U-Boot/build flow.

## Boot partition

The pack stage generates its own ext4 `boot.img` containing:

- `Image`
- `dtbs/<board>.dtb`
- `extlinux/extlinux.conf`

It also installs the same extlinux boot files into `rootfs.img` as a fallback.

## Output

Running:

```bash
sudo ./vuos pack vivid-unit
```

produces:

- `out/pack/vivid-unit/vuos-YYYYMMDD-N.img`

which can be flashed with `rkdeveloptool`.

## Boot display policy

The Vivid Unit board no longer uses a splash image.

- U-Boot is configured for text output on serial and panel (`vidconsole`).
- The kernel command line enables on-screen boot logs after hand-off.
- Plymouth/initramfs theme integration has been removed from this board.
